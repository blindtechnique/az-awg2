# Changelog

Все заметные изменения проекта. Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — по [SemVer](https://semver.org/lang/ru/).

## [2.0.2] — 2026-07-16

Исправление регрессии полного обновления и расширение возможностей бота.

### Исправлено

- **Полное обновление из бота ломало DNS и раздельную маршрутизацию** (`dns_probe_finished_bad_config`, туннелированные сайты не открывались, часть трафика шла мимо туннеля). Причина: реинтеграция слоя и пересборка клиентов выполнялись до перезагрузки, когда сервисы AntiZapret ещё не были запущены, — часть стека переинициализировалась в промежуточном состоянии. Теперь сервер перезагружается сразу после `setup.sh`, а реинтеграция слоя выполняется одноразовым systemd-юнитом уже на полностью поднятой системе. Пересборка клиентов при полном обновлении убрана — она не нужна (ключи и обфускация не меняются) и была частью проблемы.

### Добавлено

- **Проверка обновлений** (🔎 Проверить обновления): сравнивает код `setup.sh` AntiZapret и код слоя на GitHub с установленными версиями и сообщает, есть ли изменения. Списки блокировок не учитываются — они меняются постоянно и обновляются отдельно.
- **Настройки OpenVPN в редакторе параметров**: вход под одним профилем с нескольких устройств и изоляция клиентов — задаются при полном обновлении вместе с остальными параметрами.
- **Полная статистика OpenVPN**: трафик, IP, город, провайдер и история подключений — теперь копятся в той же базе, что и для WireGuard, и показываются в карточке клиента (активная сессия + накопленная история).
- **Меню подстраивается под установку**: если AntiZapret поставлен без OpenVPN или без WireGuard, соответствующие пункты в боте скрыты.

### Изменено

- После выдачи клиентских файлов (OpenVPN, стоковый WG, AmneziaWG) бот заново открывает главное меню отдельным сообщением.

## [2.0.1] — 2026-07-16

Исправления по результатам тестирования на боевых серверах.

### Исправлено

- **Split-routing ломался после полного обновления при альтернативных диапазонах IP** (`172.x`/`198.18.x`). Клиентские конфиги сохраняли прежние маршруты, не совпадавшие с пересозданными списками. Теперь раннер после `setup.sh` пересобирает конфиги клиентов слоя из свежего маршрутного файла и проверяет наличие правил раздельной маршрутизации; в отчёте появляется напоминание раздать split-клиентам новые конфиги.
- **Бот не присылал подтверждение после перезагрузки сервера.** Причина: сообщение помечалось отправленным ещё до перезагрузки. Теперь отметка ставится только после фактической отправки, с повторами на случай, когда сеть после ребута ещё не поднялась.
- **Ответ «y» при установке иногда игнорировался** (бот не ставился). При запуске через `bash <(curl …)` ввод читался из потока скрипта, а не с клавиатуры. Ввод переведён на управляющий терминал, ответы y/n нормализуются (пробелы, регистр, `yes`/`да`).
- **Debian 13: репозиторий Amnezia не подписывался** («Permission denied» при чтении keyring). Верификатор `sqv` работает от пользователя `_apt` и не мог прочитать keyring с правами `600`. Файл теперь `0644`.
- **Статистика OpenVPN не собиралась.** Добавлен разбор status-логов AntiZapret (`status-version 1`): активные сессии, туннель, IP, время подключения и трафик.

### Добавлено

- **Редактирование параметров перед полным обновлением.** Помимо «как настроено сейчас» и «всё по умолчанию» — режим «изменить параметры»: бот показывает текущие значения ключевых настроек (диапазоны клиентских и fake-IP, DNS, WARP, блокировка рекламы, маршрутизация всех доменов) и позволяет поменять любое кнопками. Изменения применяются поверх текущих ответов.

### Изменено

- Легенда в списке клиентов уточнена: раздельно обозначены split- и полнотуннельные клиенты слоя.
- README: базовый AntiZapret ставится штатной командой; обход GPG-бага через `--install-base` вынесен в примечание для тех, у кого установка падает.

## [2.0.0] — 2026-07-15

Крупная переработка: слой стал параллельным штатному AntiZapret, бот получил управление стоковыми клиентами и обновлениями сервера.

### Добавлено

- **Параллельный режим по умолчанию.** Слой AmneziaWG 2.0 поднимается на отдельных интерфейсах `antizapret-awg`/`vpn-awg`, своих подсетях (`10.29.9.x`/`10.28.9.x`) и своём UDP-порту, не изменяя штатный AntiZapret. Совместимость со сторонними админ-панелями (AdminPanelAZ и др.) без патчей.
- **Рандомный UDP-порт с закреплением.** Выбирается при установке из свободных, исключая все зарезервированные AntiZapret порты, и фиксируется в `services.env`. Флаг `--awg-ports A,V` для ручного задания.
- **Управление стоковыми WireGuard/AmneziaWG-клиентами из бота** — создание (оба туннеля разом), удаление, список. Отдаются обфусцированные `-am` конфиги. Вызовы `client.sh` сериализованы через `flock` для совместимости с админ-панелью.
- **Статистика по четырём интерфейсам** — слой AWG 2.0 и стоковые WireGuard-клиенты учитываются раздельно (колонка `origin`, метки 🅰️/🅾️). Клиенты без единого подключения отображаются корректно.
- **Меню «Обновление» в боте:**
  - «Обновить списки АнтиЗапрета» (`doall.sh`) с живым логом;
  - «Полное обновление AntiZapret» — прогон `setup.sh` с анкетой на кнопках, префлайт-проверкой новых вопросов апстрима, сохранением клиентов и автовозвратом слоя. Выполняется отдельным systemd-юнитом, переживает перезапуск бота, отчитывается после перезагрузки сервера;
  - «Обновить AWG 2.0» (`install.sh --update`);
  - «Перенастроить обфускацию» кнопками.
- **Доустановка бота отдельно** — `--install-bot [токен chat_id]`, `--remove-bot`, `--bot-token`/`--bot-admins`.
- **Миграция со старых режимов** — `--migrate` переводит `replace`/`keep` на `parallel`, сохраняя ключи клиентов.
- **Установка базы с обходом GPG-бага** — `--install-base` ставит штатный AntiZapret, обходя падение официального установщика из-за просроченного ключа OpenVPN.
- Раннер обновлений `az_setup_runner.py` (pexpect-прогон анкеты `setup.sh` + preflight).

### Изменено

- Режим работы теперь только `parallel`. Прежние `replace` и `keep` объявлены устаревшими; для установленных ранее — путь миграции через `--migrate`.
- Флаг `--keep-wireguard` больше не нужен (no-op с предупреждением) — штатный WireGuard и так остаётся активным.
- Самовосстановление после обновления AntiZapret упрощено: в `parallel` штатный WireGuard не трогается, слой возвращается через drop-in на `antizapret.service` и `awg-reintegrate.service`.
- Терминология в боте: «ванильный» → «стоковый».
- Перенастройка обфускации передаёт точные пути конфигов слоя (`antizapret-awg.conf`/`vpn-awg.conf`).

### Исправлено

- Бот не видел файлы стоковых и созданных при установке клиентов: имя файла строится по логике штатного `client.sh` (срез префиксов `antizapret-`/`vpn-` + суффикс `-(домен)`), с фолбэком на старый формат без суффикса. Затрагивало и `.ovpn`, и `-am` конфиги.
- Статистика падала с «клиент не найден» для клиентов, которые ещё ни разу не подключались.
- Перегенерация обфускации из бота не находила конфиги слоя в `parallel`-режиме.

### Зависимости

- Бот: добавлен `pexpect` (для полного обновления). Ставится в venv автоматически при `--update`.
