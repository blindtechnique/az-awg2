# Changelog

Все заметные изменения проекта. Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — по [SemVer](https://semver.org/lang/ru/).

## [2.1.0] — 2026-07-16

### Добавлено

- **Подменю «Настройки AntiZapret»** в боте — управление штатными функциями GubernievS:
  - анти-цензурный патч OpenVPN (`patch-openvpn.sh`, режимы 0/1/2);
  - включение/выключение OpenVPN DCO (`openvpn-dco.sh`);
  - правка списков маршрутизации `include-hosts`, `exclude-hosts`, `include-ips` прямо из бота (добавление и удаление строк) с напоминанием обновить списки.
  Патч и DCO показываются только при установленном OpenVPN.
- **Проверка обновлений** (🔎): сравнивает код установщика AntiZapret и код слоя на GitHub с установленными версиями и подсказывает, нужно ли обновляться. Полное обновление AntiZapret выполняется штатной командой в терминале — бот показывает её. Списки блокировок в проверку не входят.
- **URI `vpn://` и QR для стокового WireGuard**: при создании и скачивании стокового клиента бот отдаёт для обоих туннелей обфусцированный конфиг, QR и ссылку для приложения Amnezia.
- **Полная статистика OpenVPN**: трафик, IP, город, провайдер и история подключений копятся в общей базе; в карточке клиента — активная сессия и накопленные данные.
- **Меню подстраивается под установку**: если AntiZapret поставлен без OpenVPN или без WireGuard, соответствующие пункты скрыты.

### Изменено

- После выдачи клиентских файлов бот заново открывает главное меню отдельным сообщением.
- Легенда в списке клиентов уточнена: раздельно обозначены split- и полнотуннельные клиенты слоя.

### Исправлено

- Бот не видел файлы стоковых и созданных при установке клиентов (имя файла строится по логике штатного `client.sh`).
- Статистика не находила клиентов, которые ещё ни разу не подключались.
- Перегенерация обфускации из бота не находила конфиги слоя в parallel-режиме.
- Debian 13: репозиторий Amnezia не подписывался из-за прав на keyring (теперь `0644`).
- Ответ «y» при установке через `bash <(curl …)` иногда игнорировался — ввод переведён на управляющий терминал, ответы нормализуются.

## [2.0.0] — 2026-07-14

### Добавлено

- **Параллельный режим по умолчанию.** Слой AmneziaWG 2.0 работает поверх штатного AntiZapret, не изменяя его: свои интерфейсы `antizapret-awg`/`vpn-awg`, свои подсети и UDP-порт. Совместимость со сторонними админ-панелями без патчей.
- **Рандомный UDP-порт с закреплением**, исключающий зарезервированные AntiZapret порты; ручное задание через `--awg-ports`.
- **Управление стоковыми WireGuard/AmneziaWG-клиентами из бота** — создание, удаление, список (через `client.sh` под `flock`).
- **Статистика по всем интерфейсам** — слой AWG 2.0 и стоковые клиенты учитываются раздельно; трафик, онлайн, история подключений с гео.
- **Кнопки обновления в боте**: обновление списков (`doall.sh`), обновление кода слоя (`install.sh --update`), перенастройка обфускации.
- **Доустановка и удаление бота** отдельными флагами: `--install-bot`, `--remove-bot`, `--bot-token`, `--bot-admins`.
- **Миграция со старых режимов** `replace`/`keep` на `parallel` через `--migrate` с сохранением ключей клиентов.
- **Установка базы с обходом GPG-бага** через `--install-base` (для случаев, когда официальный установщик падает из-за просроченного ключа OpenVPN).

### Изменено

- Режим работы теперь только `parallel`; прежние `replace`/`keep` объявлены устаревшими (путь миграции — `--migrate`).
- Самовосстановление слоя после обновления AntiZapret: в `parallel` штатный WireGuard не трогается, слой возвращается через drop-in и `awg-reintegrate.service`.
- Терминология в боте: «ванильный» → «стоковый».

## [1.0.0] — 2026-07-13

Первый публичный релиз.

### Добавлено

- Слой настоящего AmneziaWG 2.0 поверх AntiZapret с собственным транспортом и обфускацией (в ранних режимах `replace`/`keep`).
- Раздельная маршрутизация (AntiZapret) и полный VPN на AmneziaWG-слое.
- Настраиваемая обфускация: пресеты интенсивности, шаблоны мимикрии (QUIC/TLS/DNS/VoIP), выбор MTU и домена.
- Telegram-бот: управление клиентами AmneziaWG и OpenVPN, статистика, бэкап и восстановление.
- Клиентские артефакты: `.conf`, QR и ссылка `vpn://` для приложения Amnezia.
- Временные клиенты с автоудалением по TTL.
