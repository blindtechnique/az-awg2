# Changelog

Все заметные изменения проекта. Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — по [SemVer](https://semver.org/lang/ru/).

## [2.0.0] — 2026-07-15

Крупная переработка: слой стал параллельным штатному AntiZapret, бот получил управление стоковыми клиентами и обновлениями сервера.

### Добавлено

- **Параллельный режим по умолчанию.** Слой AmneziaWG 2.0 поднимается на отдельных интерфейсах `antizapret-awg`/`vpn-awg`, своих подсетях (`10.29.9.x`/`10.28.9.x`) и своём UDP-порту, не изменяя штатный AntiZapret. Совместимость со сторонними админ-панелями (AdminPanelAZ и др.) без патчей.
- **Рандомный UDP-порт с закреплением.** Выбирается при установке из свободных, исключая все зарезервированные AntiZapret порты, и фиксируется в `services.env`. Флаг `--awg-ports A,V` для ручного задания.
- **Управление стоковыми WireGuard/AmneziaWG-клиентами из бота** — создание (оба туннеля разом), удаление, список. Отдаются обфусцированные `-am` конфиги. Вызовы `client.sh` сериализованы через `flock` для совместимости с админ-панелью.
- **Статистика по четырём интерфейсам** — слой AWG 2.0 и стоковые WireGuard-клиенты учитываются раздельно (колонка `origin`, метки 🅰️/🅾️). Клиенты без единого подключения отображаются корректно.
- **Меню «Обновление» в боте:**
  - «Обновить списки АнтиЗапрета» (`doall.sh`) с живым логом;
  - «Полное обновление AntiZapret» — прогон `setup.sh` с анкетой на кнопках, префлайт-проверкой новых вопросов апстрима, сохранением клиентов и автовозвратом слоя. Выполняется отдельным systemd-юнитом, переживает перезапуск бота, отчитывается после перезагрузки сервера;
  - «Обновить AWG 2.0» (`install.sh --update`);
  - «Перенастроить обфускацию» кнопками.
- **Доустановка бота отдельно** — `--install-bot [токен chat_id]`, `--remove-bot`, `--bot-token`/`--bot-admins`.
- **Миграция со старых режимов** — `--migrate` переводит `replace`/`keep` на `parallel`, сохраняя ключи клиентов.
- **Установка базы с обходом GPG-бага** — `--install-base` ставит штатный AntiZapret, обходя падение официального установщика из-за просроченного ключа OpenVPN.
- Раннер обновлений `az_setup_runner.py` (pexpect-прогон анкеты `setup.sh` + preflight).

### Изменено

- Режим работы теперь только `parallel`. Прежние `replace` и `keep` объявлены устаревшими; для установленных ранее — путь миграции через `--migrate`.
- Флаг `--keep-wireguard` больше не нужен (no-op с предупреждением) — штатный WireGuard и так остаётся активным.
- Самовосстановление после обновления AntiZapret упрощено: в `parallel` штатный WireGuard не трогается, слой возвращается через drop-in на `antizapret.service` и `awg-reintegrate.service`.
- Терминология в боте: «ванильный» → «стоковый».
- Перенастройка обфускации передаёт точные пути конфигов слоя (`antizapret-awg.conf`/`vpn-awg.conf`).

### Исправлено

- Бот не видел файлы стоковых и созданных при установке клиентов: имя файла строится по логике штатного `client.sh` (срез префиксов `antizapret-`/`vpn-` + суффикс `-(домен)`), с фолбэком на старый формат без суффикса. Затрагивало и `.ovpn`, и `-am` конфиги.
- Статистика падала с «клиент не найден» для клиентов, которые ещё ни разу не подключались.
- Перегенерация обфускации из бота не находила конфиги слоя в `parallel`-режиме.

### Зависимости

- Бот: добавлен `pexpect` (для полного обновления). Ставится в venv автоматически при `--update`.
